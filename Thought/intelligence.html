01a neuron is just a weighted opinion

One neuron does embarrassingly little: it takes some inputs, multiplies each by a weight (how much it cares), adds a bias (its general mood), and squashes the result between 0 and 1. That final number is how strongly it fires.

02a network is neurons voting on each other

Your drawing becomes a 28×28 grid — 784 brightness values. Those feed 64 hidden neurons, and each of those feeds 10 output neurons, one per digit. The loudest output wins.

Nobody told the hidden neurons what to look for. These are actual weights from the network on this page — each tile is one hidden neuron's 784 weights arranged back into a 28×28 image. Cyan pixels excite the neuron, pink pixels inhibit it. It invented these stroke-detectors by itself:

03learning is rolling downhill

Training is a game of golf in 50,890 dimensions. The loss measures how wrong the network is. Each weight asks one question: if I nudge myself, does the loss go down? Then every weight takes a small step downhill at once. That's gradient descent — calculus finds the slope, repetition does the rest.

Here's one weight rolling down its loss bowl. Crank the learning rate past 0.9 and watch it overshoot back and forth — the classic failure of stepping too far.

05the same recipe, full size

Everything you just played with scales up. The network on this page was trained with the exact loop below — forward pass, measure the loss, backpropagate, nudge the weights, repeat 840,000 times. GPT does fundamentally this, with more zeros on the end.

the whole thing in PyTorch 
import torch, torch.nn as nn
from torchvision import datasets, transforms

train = datasets.MNIST('data', train=True, download=True,
                       transform=transforms.ToTensor())
loader = torch.utils.data.DataLoader(train, batch_size=32, shuffle=True)

model = nn.Sequential(
    nn.Flatten(),
    nn.Linear(28 * 28, 64),  # 784 pixels -> 64 hidden neurons
    nn.ReLU(),
    nn.Linear(64, 10),       # 64 hidden -> 10 digits
)

opt = torch.optim.SGD(model.parameters(), lr=0.05)
loss_fn = nn.CrossEntropyLoss()

for epoch in range(14):
    for x, y in loader:
        opt.zero_grad()
        loss = loss_fn(model(x), y)   # how wrong are we?
        loss.backward()               # which way is downhill?
        opt.step()                    # take the step

That's the secret, in full: a pile of weighted opinions, a way to measure wrongness, and the patience to roll downhill 840,000 times. You just taught one with your own hands.
